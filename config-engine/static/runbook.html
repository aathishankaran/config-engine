Config Engine — User Runbook

A visual dataflow interface builder for mainframe transformation pipelines. Design, edit, and validate COBOL/JCL dataflow interfaces using the drag-and-drop canvas or the built-in JSON editor.

Current Version Flask · Python Internal Use Only Updated Mar 2026
1

Overview

Config Engine is a browser-based tool for creating and managing dataflow interfaces — structured JSON files that describe how mainframe batch data moves through transformation steps (Inputs → Transformations → Outputs).

Interfaces are used by the Dataflow Engine to execute real data pipelines against COBOL/fixed-width/Parquet source files. Config Engine lets engineers design these pipelines visually without hand-editing complex JSON.

Dual Interface: Every interface can be edited both as a visual node graph (Studio Canvas) and as raw JSON (JSON Editor). Changes in either view are kept in sync.

Prerequisites

Requirement Version Notes
Python 3.10+ Virtual environment (.venv) recommended
Flask 3.x Listed in requirements.txt
Browser Chrome 100+ / Safari 16+ / Edge 100+ JavaScript must be enabled
Java (optional) JDK 11 Required only for COBOL parsing features

Java Version Note: If you see UnsupportedOperationException: getSubject is not supported, your JVM version is > 17. Switch to JDK 11 or add -Djava.security.manager=allow to your JVM arguments.

Launching the App

  1. Clone / navigate to the project

    cd cloude-engine/config-engine

  2. Activate the virtual environment

    # macOS / Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate

  3. Install dependencies (first run only)

    pip install -r requirements.txt

  4. Start the Flask server

    python app.py

    The server starts on http://localhost:5000 by default.

  5. Open the app in your browser

    Navigate to http://localhost:5000

5

Load an Interface

  1. Find your config in the left panel

    All saved .json configs in the configs/ folder appear in the list. Use the search box to filter by name.

  2. Click the config name

    The canvas loads the dataflow pipeline automatically. Nodes and connections are rendered in the Studio Canvas.

  3. Fit the view

    Press F or click the Fit button in the toolbar to fit all nodes into view.

  4. Or load from toolbar

    Click Load in the top toolbar and pick any config file from the server.

Tip: After loading, the toolbar shows Test, Download, Rename, and Delete action buttons for the active interface.

6

Studio Canvas

The canvas is a pan-and-zoom workspace where you build your dataflow by connecting nodes.

Adding Nodes

  1. Drag from the Component Palette

    In the right panel, drag any component (Input, Select, Filter, etc.) onto the canvas.

  2. Name the node

    A dialog prompts for a node name. Enter a unique identifier (e.g., TXNIN).

Connecting Nodes

  1. Switch to Connect mode

    Press C or click Connect in the toolbar. The cursor changes to a crosshair.

  2. Draw a connection

    Click the source node, then click the target node. An arrow appears.

  3. Return to Select mode

    Press V or click Select to go back to the default mode.

Node Properties

Click any node on the canvas to open its Properties Panel on the right side.

  1. Click a node

    The node highlights and the Properties Panel slides in from the right.

  2. Edit fields

    Each node type has different fields: Input nodes have format, path, schema; Join nodes have left/right source, join key; Filter has condition, etc.

  3. Click Apply

    Press the Apply button to save changes to the node and auto-sync to the server.

  4. Switch nodes safely

    When you click another node, unsaved changes are auto-saved silently — you won't lose edits.

Auto-save on switch: Clicking a different node automatically applies your current edits first, so you never lose changes when navigating between nodes.

8

JSON Editor

Switch to the full-screen JSON Editor for direct JSON manipulation. The editor uses the JSONEditor library with a VS Code–inspired dark theme.

  1. Open the JSON Editor

    Click JSON Editor in the center mode bar (appears after loading a config).

  2. Navigate the tree

    Expand / collapse sections using the triangle arrows. The breadcrumb at the top shows your position.

  3. Switch modes

    Use the Tree / Code mode toggle in the editor toolbar to switch between visual tree view and raw JSON code view.

  4. Apply changes

    Click Apply to Canvas (green button, top-right of the editor) to push JSON changes back to the visual canvas.

  5. Close without applying

    Click Close to dismiss the editor and return to the canvas without applying changes.

Color Coding: Keys are blue, string values are orange, numbers are green, booleans are blue-grey.

9

Import & Export JSON

Importing a JSON file

  1. Click "Import JSON" in the toolbar

    A modal dialog opens with a file upload area and a text paste area.

  2. Upload or paste JSON

    Either drag a .json file onto the upload area, or paste raw JSON into the text field.

  3. Click Import

    The pipeline is validated and rendered on the canvas immediately.

Exporting a JSON file

  1. Click "Export JSON" in the toolbar

    The current canvas interface is downloaded as a .json file.

Download button: After loading a config, the Download button (red, in toolbar) also exports the current interface file directly.

10

Create New Interface

  1. Click "New" in the toolbar

    If a config is loaded, you are prompted to save changes first.

  2. Enter a config name

    Type a name in the config name field (e.g., MY_PIPELINE.json).

  3. Drag components from the right palette

    Build your pipeline by adding Input, Transformation, and Output nodes.

  4. Connect nodes in Connect mode (C)

    Draw arrows from source to target nodes to define data flow.

  5. Click Save

    Press Save in the toolbar. The config is saved to configs/ on the server and appears in the left list.

Component Reference

ComponentTypeDescriptionKey Properties
Input Source Read data from CSV, Parquet, COBOL fixed-width, or JCL files format, path, schema, header
Output Sink Write results to Parquet, CSV, or COBOL files format, path, partition
Oracle Write Sink Load data into Oracle via SQL*Loader with Vault-based credentials host, table, load_mode, vault_path
Validate Quality Data quality checks — route valid rows to output, invalid rows to error path fail_mode, validate_rules[], validated_path, error_path
Mapping Transform Field-level mapping and renaming between source and target schemas mappings[] with source, target, transform
Select Transform Map, rename, or compute columns (SQL-like expressions) columns[] with alias and expr
Filter Transform Row-level filtering using a boolean condition string condition
Join Transform Merge two datasets on a common key left, right, on, how
Aggregate Transform Group and aggregate (SUM, COUNT, AVG, etc.) group_by[], aggregations[]
Union Transform Stack two datasets vertically (UNION ALL) inputs[]
Custom Transform Raw JSON logic for advanced or unsupported operations logic (raw JSON object)

Troubleshooting

ProblemLikely CauseFix
App not loading at localhost:5000 Flask server not running Run python app.py in the config-engine directory
UnsupportedOperationException: getSubject Java version > 11 Use JDK 11 or add -Djava.security.manager=allow to JVM args
Config not saving Empty config name field Enter a name in the config name text box before clicking Save
JSON Editor not opening No config loaded Load or create an interface first; the mode bar appears only when an interface is active
Nodes disappeared after Apply Invalid JSON in editor Check the JSON editor for red validation errors before applying
Can't connect nodes Not in Connect mode Press C or click Connect in the toolbar first
